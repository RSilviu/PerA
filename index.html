Abstract

PerA (Personal Web Assistant) is a web application intended to help people manage their activities.

Users will be able to schedule activities, connect with other users and follow them.

Technologies

  • AJAX stands for Asynchronous JavaScript And XML. In a nutshell, it is the use of the XMLHttpRequest object to communicate with servers. It can send and receive information in various formats, including JSON, XML, HTML, and text files. AJAX’s most appealing characteristic is its "asynchronous" nature, which means it can communicate with the server, exchange data, and update the page without having to refresh the page.

  • XAMPP stack Many people know from their own experience that it's not easy to install an Apache web server and it gets harder if you want to add MariaDB, PHP and Perl. The goal of XAMPP is to build an easy to install distribution for developers to get into the world of Apache.

  • Google Maps Javascript API lets you customize maps with your own content and imagery for display on web pages and mobile devices.

  • Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. We also used Github for hosting the source code. We divided the project into small tasks and created one branch per feature, starting with the front-end part. When features were completed, the corresponding branches would be merged into the master branch. For the server side, we went for a different approach. We pushed to branches called 'dev' and 'latest', the latter having most recent changes.

Architecture

Design Details

The User Interface of the application is built using HTML5 and CSS3. Some UI parts are dinamically generated from Javascript.

Conclusion & Future considerations

PerA has been created to ease up the management of activities by providing an intuitive platform that uses simple technologies to enhance the way of visualizing the relationships between various entities in the context of real life. The application can be extended and improved in different ways by conceiving new mechanisms and architectural ideas.

All of these can be integrated in the current version of the design resulting in a modern, evolved PerA.
